# AI 资源层

> **层级**: L4 - AI 资源（How to Work）  
> **角色**: 全团队  
> **本质**: 人机协作方法论

## 目录结构

```
04-AI-assets/
├── AI-coding/           # AI 编码资源
│   ├── rules/          # 编码规则与约束
│   ├── skills/         # AI 技能
│   ├── workflows/      # 开发工作流
│   └── scripts/        # 工具脚本
└── skills/             # 团队共享技能
```

## 使用方式

### Rules（规则）
定义项目的架构原则、编码规范、测试策略等强约束。

### Skills（技能）
定义可复用的 AI 协作技能，明确输入、处理、输出。

### Workflows（工作流）
定义标准化的开发流程，串联多个技能。

## 设计原则

每个 Skill/Workflow 应明确定义：
- **输入**：需要什么？（优先 AI-friendly）
- **处理**：怎么做？
- **输出**：产出什么？（AI-friendly & User-friendly）
